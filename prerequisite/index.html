  3. most important mvtest APIs
    • ExpectShell API for interactive testing
    • RemoteMachine API for running commands on remote host
    • run_cmd API for running commands locally
    • get_status_output API if command exit status and output is needed.
  4. custom logging levels implemented using custom_logger
    mvtest uses following custom logging levels
    • log.command for COMMAND logging
    • log.texitcode for EXIT_CODE logging
    • log.output for OUTPUT logging
    • log.tpass for PASS logging
    • log.tfail for FAIL logging
