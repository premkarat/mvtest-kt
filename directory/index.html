  • mvtest directory structure overview
  1. apis

    kernutils contains commonly used functions for kernel related testing.
    netutils contains commonly used functions for network related testing
    utils contains general purpose utility APIs. This file has ExpectShell, RemoteMachine, run_cmd etc.. API's

  2. conftest.py

    One of the usecases of conftest is to implement a local plugin. A plugin contains one or multiple hook functions. writing hook functions explains how to write hooks in pytest. We use pytest_configure hook to enhance the default HTML report with additonal environmental details. How to enhance HTML reports section in pytest-html page has details.

  3. docs

    This folder contains all the pieces required for documentation generation. mvtest uses Sphinx for HTML documentation generation. Initial mvtest documentation was generated by following this Sphinx tutorial. The most important files in this directory are

    1. Makefile used for generating docs using make command. Makefile options section explains customizing Makefile.

    2. conf.py controls the basics of how sphinx runs when you run a build. For example, mvtest uses readthedocs theme by specifying the html_theme config parameter. Basic Configuration tutorial explains how to modify the conf.py file

    3. make command is used for building the documentation. list of available builders shows the options that can be used with make command. mvtest users make html to generate mvtest documentation.

  4. license.txt

    Has MIT license information

  5. pytest.ini

    This is the configuration file used by pytest. Basic test configuration can be configured usign this file. Various configurations parameters for pytest.ini can be found here

  6. README.md

    README file in Markdown format required for github.

  7. requirements.txt

    Has a list of dependency packages, along wth supported versions, required for mvtest to work. Detailed documentation of requirements.txt file can be found here

  8. suites

    This folder has all the tests organized based on various profiles supported by MontaVista Linux distributions.
